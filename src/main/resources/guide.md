# 研发实践题:末端驿站包裹管理系统

## 任务背景
在快递物流的“最后一百米”环节，末端驿站（如菜鸟驿站、丰巢等）承担着包裹的接收、存放和交付工作。
为了提升驿站⼩哥的操作效率，我们需要开发⼀个轻量级的包裹管理原型系统。

## 技术要求
后端：Java (Spring Boot) + MyBatis。遵循经典的Controller-Service-Mapper三层架构，设计RESTful API供前端调用。
前端：技术栈不限（Vue, React, 或原生HTML/JS均可），需具备基本的交互能力。
数据库：MySQL。

## 功能需求
1. 包裹入库逻辑：开发包裹入库功能。输入项包括：运单号、收件人手机号、快递公司、货架位置。
2. 包裹查询与出库管理：支持按手机号查询该名下所有“代取件”包裹。点击“确认取件”后包裹状态变为“已取件”，并记录出库时间。
3. 逾期滞留逻辑:在包裹列表页面，对于入库时间超过 48 小时的包裹，需有显著的“滞留”标识(如红色背景或告警标签)。
4. 测试需求: 为本系统设计5~10条测试用例,全方位测试系统的正常功能与异常场景。
